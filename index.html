Overview

What is Agentic AI?

"Agentic AI" refers to AI that can take actions on your behalf‚Äîreading files, writing code, running programs, and completing multi-step tasks autonomously. Instead of just answering questions, Claude CLI can actually do things: analyze your data, build tools, automate workflows, and more.

You describe what you want in plain English. Claude figures out the steps, writes any necessary code, runs it, and delivers results. You guide the process with feedback, but Claude handles the technical details.

üí°
No coding experience required. You'll learn to describe tasks clearly and review results‚ÄîClaude handles the programming.

Phase 1: Foundation Setup

‚è±Ô∏è 30-45 minutes Not started

Install the basic tools and get comfortable with VS Code's layout. You can stop after this phase and return later.

Step 1.1: Install VS Code

  1. Open your browser and go to: code.visualstudio.com
  2. Click the big blue "Download for Mac" button
  3. Wait for the download, then double-click the .zip file to extract it
  4. Drag "Visual Studio Code" to your Applications folder
  5. Run the downloaded installer (VSCodeUserSetup-x64-x.xx.x.exe)
  6. Accept the license agreement and click Next
  7. IMPORTANT: On "Select Additional Tasks," check ALL boxes, especially "Add to PATH"
  8. Open VS Code from Applications
üñºÔ∏è VS Code Welcome Screen
‚úì
Success looks like: A window opens with "Welcome" tab. You'll see an interface with icons on the left side.

Step 1.2: Learn the VS Code Layout

Spend 5 minutes understanding the interface‚Äîit'll make everything easier.

Activity Bar Far left icons. The files icon (Explorer) is most important.
Sidebar Shows your folders and files, like Finder/File Explorer.
Editor Area Where you view and edit files. Files open in tabs.
‚≠ê Terminal THE IMPORTANT ONE. You'll interact with Claude here. Press Ctrl+` to toggle.

üéØ Try it now:

  1. Press Ctrl+` (backtick key, left of 1). Terminal appears at bottom.
  2. Press Ctrl+` again. Terminal hides.
  3. Press Ctrl+` once more. Click in the Terminal area.
üí°
The Terminal is your command center. Ctrl+` is the most important shortcut you'll learn.

Step 1.3: Install Homebrew

Homebrew is a package manager that makes installing developer tools easy on Mac.

  1. Open VS Code with Terminal visible (Ctrl+`)
  2. Copy and paste this command, then press Enter:
Terminal 
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  1. Enter your Mac password when prompted (you won't see characters‚Äîthat's normal)
  2. Press Enter when asked to continue
  3. Wait 2-5 minutes for installation
  4. CRITICAL: Look for "Next steps" at the end and run those commands!
‚ö†Ô∏è
Don't skip the "Next steps" commands! They look something like:
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile

Verify Homebrew works:

Terminal 
brew --version
‚úì
Success: You see "Homebrew 4.x.x"
üîß Troubleshooting

"command not found" - You missed the "Next steps." Close Terminal, reopen, and run the commands shown at the end of Homebrew installation.

Step 1.4: Install Python

Use Homebrew to install Python:

Terminal

  2. Click the yellow "Download Python 3.x.x" button
  3. Run the downloaded installer
  4. CRITICAL: Check "Add Python to PATH" at the bottom!
  5. Click "Install Now"
‚ö†Ô∏è
If you miss "Add Python to PATH," uninstall and reinstall with that box checked.

Verify installation (wait for install to finish):

Terminal

‚úì
Success: You see "Python 3.12.x" or "Python 3.13.x"
üí°
On Mac, always use python3 (not just python).

Step 1.5: Install Git

Step 3.2: Create .claude Configuration

The .claude folder gives Claude context about your project. Claude reads CLAUDE.md automatically when you start a session.

Terminal 
mkdir -p .claude

Start Claude and ask it to create the configuration files:

Terminal 
claude
Say to Claude 
Create these files for my project:

1. .claude/CLAUDE.md with:
   - Project name: My First Project
   - Description: A workspace for AI-assisted analysis and automation
   - Virtual environment: venv/ (activate with "source venv/bin/activate")
   - A Session Start section reminding to read AI_CODING_GUIDELINES.md

2. .claude/AI_CODING_GUIDELINES.md with rules:
   - State approach before making changes
   - Only modify what I ask
   - Ask if unsure rather than guessing

3. .claude/SESSION_LOG.md with a template for logging sessions

Daily Workflow

Starting a Session

  1. Open VS Code
  2. Open project folder: File ‚Üí Open Folder
  3. Open Terminal: Ctrl+`
  4. Activate venv: source venv/bin/activate
  5. Start Claude: claude
  6. Describe your task

Example Tasks

"Read the sales.xlsx file and tell me which products had declining sales quarter over quarter. Show me a summary table."

‚Üí Claude reads the file, analyzes trends, and displays results.

"Rename all PDF files in the invoices folder using the date found in each filename. Format: YYYY-MM-DD_invoice.pdf"

‚Üí Claude examines files, writes a script, and renames them.

"Create a script that pulls data from this API endpoint and saves it to a CSV file daily."

‚Üí Claude builds the script and explains how to schedule it.

"This script is throwing an error: [paste error]. What's wrong and how do I fix it?"

‚Üí Claude diagnoses the issue and provides a fix.

Ending a Session

  1. Update log: Tell Claude "Update SESSION_LOG with what we did"
  2. Exit Claude: /exit
  3. Save to GitHub:
Terminal 
git add . && git commit -m "Description of changes" && git push

"Command not found"

The tool isn't installed or isn't in your PATH.

  • Close and reopen VS Code
  • Make sure you ran the "Next steps" after Homebrew
  • Try reinstalling the tool
Terminal seems frozen

A command is running or stuck.

  • Press Ctrl+C to cancel
  • Close Terminal and open new one (Ctrl+`)
  • Restart VS Code
Claude made a mistake

This is normal! AI isn't perfect. Just tell Claude:

  • "That's not right. Undo that change."
  • "That broke something. Here's the error: [paste]. Fix it."
  • "Try a different approach."
Missing packages error

You probably forgot to activate your virtual environment.

Run: source venv/bin/activate

Look for (venv) at the start of your prompt.

PowerShell execution policy error

Run this once:

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
